## 2. 存储模式选择

系统支持两种文件存储后端，可在「应用管理 → 服务配置 → 服务功能」的「存储模式」下拉中切换，保存即生效：

- **七牛云对象存储**（默认）：依赖外部 OSS 服务，上传下载速度稳定，无存储上限限制，有一定免费额度。
- **本机磁盘**：文件直接落到服务器 `upload` 目录，适合内网、单机或轻量部署。

:::tip 注意！
如果服务器带宽不足，建议使用七牛云对象存储。或限制上传文件的大小，避免文件过大占用机器资源导致服务崩溃。
:::

切换是安全的：每条文件记录会带有上传时的存储类型（`local` / `qiniu`），后端按记录独立路由，**切换不会影响历史文件的访问，也不需要做存量迁移**。

### 2.1 本地存储配置

切到「本机磁盘」后，可在同一表单调整「单文件最大上传大小（MB）」（默认 `500`），保存即生效。

- **存储目录**：服务进程工作目录下的 `upload/`，即 `${cwd}/upload`，文件按 `upload/easypicker2/{taskKey}/{hash}/{filename}` 组织；管理面板「概览」会展示当前 `upload` 目录的绝对路径与占用空间。

- **磁盘维护建议**：
  - 定期通过 `du -sh upload/` 检查容量。
  - `.gitignore` 已忽略该目录，不会被提交到仓库。
  - 备份建议直接备份 `upload/easypicker2/` 子目录；迁移服务器时把整个 `upload` 目录原样同步到新机即可。

## 5. 邮箱配置（可选）

系统支持通过 SMTP 发送邮件。配置完成后可覆盖如下业务场景：

- **邮箱验证码登录**：登录页支持"邮箱 + 验证码"登录方式。
- **注册时绑定邮箱**：开启「强制绑定邮箱」后，注册必须完成邮箱验证。
- **找回密码**：通过邮箱验证码重置登录密码。
- **用户绑定邮箱**：个人中心可绑定/换绑/解绑邮箱。
- **文件提交通知**：任务所有者开启提醒后，提交/撤回文件会收到邮件。
- **服务错误告警**：服务运行抛出未捕获错误时，向「服务告警收件邮箱」发送告警邮件。

在「服务配置」页面打开「邮箱配置」Tab，里面分为两个区块。

### 5.1 SMTP 配置

填写发信邮箱的 SMTP 服务信息：

- `SMTP 主机`：例如 QQ 邮箱为 `smtp.qq.com`
- `端口`：常用 `465` 或 `587`
- `是否 SSL`：`465` 通常开启，`587` 通常关闭
- `发信账号`：完整邮箱地址
- `密码/授权码`：邮箱服务商生成的 SMTP 授权码，通常不是登录密码
- `发件人邮箱`：建议与发信账号保持一致
- `发件人显示名`：邮件里展示的发件人名称

以 QQ 邮箱为例，通常配置为：

```text
SMTP 主机：smtp.qq.com
端口：465
是否 SSL：开启
发信账号：xxx@qq.com
密码/授权码：QQ 邮箱生成的授权码
发件人邮箱：xxx@qq.com
```

:::warning
QQ、网易、企业邮箱等服务商一般要求先在邮箱后台开启 SMTP/IMAP/POP 服务，并使用「授权码」登录 SMTP。
如果测试时报 `535 Login fail`，优先检查授权码、SMTP 服务开关、SSL/端口和登录频率限制。
:::

### 5.2 其它邮箱功能配置

- `邮箱验证码`：默认关闭。开启后且 SMTP 配置完整时，支持邮箱验证码登录、注册绑定邮箱、找回密码和用户绑定邮箱。
- `强制绑定邮箱`：默认关闭。开启后且 SMTP 配置完整时，注册页会把邮箱输入放在表单最前面；输入正确邮箱后展示邮箱验证码输入项，并要求完成邮箱验证。
- `服务告警收件邮箱`：服务运行异常时接收告警邮件。多个邮箱可用逗号、分号或空格分隔。
- `每日发信上限`：全局每日邮件发送上限。设置为 `0` 表示不限制。

### 5.3 测试邮箱

邮箱配置 Tab 右上角提供「测试邮箱」按钮。填写接收邮箱后可测试：

- SMTP 基础连通
- 邮箱验证码模板
- 文件提交通知
- 服务错误告警
- 每日发信上限提示

测试邮件会真实发送，也会计入每日发信上限。刚修改 SMTP 或邮箱功能配置后，请先保存再测试。
